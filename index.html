Welcome to the documentation for the Personalized-Scheduler-and-Project-Management system. This system allows users to configure and visualize personal schedules and project management timelines.

Overview

This project focuses on creating a personalized scheduler and project management tool to assist users in organizing their tasks effectively using interactive widgets, dynamic schedules, and data visualization techniques.

Features

  • Interactive schedule generation based on user input.
  • Data storage and manipulation using CSV files.
  • Integration with Google Colab for visualization.
  • Customizable time blocks for activities.

Setup

To run the project, ensure the following:

  • Python 3.x installed.
  • Libraries like Pandas, Numpy, and Matplotlib are available.
  • Use Google Colab for optimal widget-based interactivity.

Personal Schedule

personal_schedule()

Displays the personal schedule module where users can configure their daily schedule. The user is prompted to input a time block interval, such as "1 hour," for each block in the schedule. Additionally, rest hours, like "11:00 PM to 7:00 AM," and meal times such as breakfast at "8:00 AM" are inputted. Once the user provides this information, the module displays the schedule for the day with designated meal and rest times.

Project Schedule

project_schedule()

Displays the project schedule module, where users can configure and visualize tasks for a project. Users can add tasks with details such as the task name, start date, duration, the person assigned to the task, and completion status. The project schedule is then displayed, and users can generate a Gantt chart to visualize task timelines and their progress.

Utility Functions

Constants

  • TIME_BLOCK_INTERVAL: Specifies the duration for each block in the personal schedule. Default is "1 hour" but can be adjusted to "30 minutes", "45 minutes", etc.
  • REST_HOURS: A tuple defining the start and end time for the rest period (e.g., ("12:00 AM", "6:00 AM")).
  • MEAL_TIMES: A dictionary specifying the times for breakfast, lunch, and dinner.

get_closest_time_idx(timetable_df, target_time)

This helper function takes a timetable DataFrame and a target time as input. It returns the index of the closest time in the timetable that matches the given target time. This is useful for assigning activities to the correct time slots.

load_and_display_options()

Loads the saved timetable and project data from CSV files. It displays two buttons that allow users to view their daily or weekly timetable. If no files are found, an error message is displayed.

generate_daily_timetable(weekly_events, projects)

Generates a daily timetable by first setting up the structure for the day's schedule, marking rest hours, and meal times. It then populates the timetable with events and assigns tasks to available free time. The final timetable is displayed in a bar chart format.

generate_weekly_timetable(weekly_events, projects)

Generates a weekly timetable, displaying repeating events and tasks due on each day of the week. It compares event days and deadlines with the current date to display relevant information for the user.

plot_timetable(timetable_df, today_day)

Plots the timetable for a given day using a horizontal bar chart. The different activities (e.g., "Free Time", "Rest", "Breakfast") are represented with different colors, and the timetable is displayed with times on the vertical axis.

Project Management

manage_projects()

This function creates or loads project files using a graphical user interface (GUI). Users can manage their projects by either creating a new project or loading an existing project file. The project is saved in CSV format, with columns such as "Task", "Start", "Duration", "Assigned To", "Priority", "Description", "Dependencies", "Deadline", "Milestone", "Estimated Hours", "Actual Hours", "Comments", and "Completed". If a project is created, an empty file with these headers is generated.

load_tasks_from_file()

This function loads the tasks from the active project file. If the project file exists, it is loaded into a DataFrame, and the task dropdown menu and progress bar are updated. If the project file does not exist or is invalid, an empty DataFrame is initialized.

update_task_dropdown()

This function updates the task dropdown menu with available task names from the project file. The dropdown will allow users to select existing tasks or add new ones. The dropdown options are dynamically updated whenever a task is added or loaded.

on_task_select(change)

This function is triggered when a task is selected from the dropdown. It will either autofill the task details if an existing task is selected or clear the task fields if the "Add New Task" option is chosen.

autofill_task_details(task_details)

This function automatically fills in the details of a task in the form fields when a task is selected from the dropdown. The fields populated include task name, start date, duration, assigned person, priority, description, dependencies, deadline, milestone, estimated hours, actual hours, and comments.

calculate_deadline(task_details)

This function calculates the deadline of a task by adding the task duration (in days) to the start date. It is used to auto-complete the deadline for new tasks.

clear_task_fields()

This function clears the input fields, allowing the user to start fresh when adding a new task. It resets all form inputs to their default values.

save_task_data(b)

This function saves the task data to the project file. If the task already exists, its details are updated; otherwise, the task is added as a new entry in the project file. After saving, the tasks are reloaded, and the progress bar is updated.

update_progress_bar()

This function updates the progress bar based on the completion status of tasks. It calculates the percentage of completed tasks and updates the progress bar accordingly.

Task Management Widgets

The following widgets allow users to input task details, select task dependencies, and manage progress:

  • task_dropdown: Dropdown for selecting a task.
  • task_name_input: Input field for entering the task name.
  • start_date_input: DatePicker for selecting the task start date.
  • duration_input: Integer input for entering the task duration in days.
  • assigned_to_input: Dropdown for selecting the person assigned to the task.
  • task_priority_input: Dropdown for selecting the task priority (Low, Medium, High).
  • task_description_input: Textarea for entering a task description.
  • task_dependency_input: SelectMultiple widget for choosing task dependencies.
  • task_deadline_input: DatePicker for selecting the task deadline.
  • milestone_input: Checkbox for marking the task as a milestone.
  • estimated_hours_input: Integer input for entering estimated hours for the task.
  • actual_hours_input: Integer input for entering actual hours spent on the task.
  • task_comments_input: Textarea for entering additional comments about the task.
  • save_button: Button for saving the task data.
  • progress_bar: Progress bar for displaying task completion status.
